📚 ¿Cómo Funciona Esta Vulnerabilidad?

🚨 Código Vulnerable:

query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"

🔍 Problema:

  • Los datos del usuario se concatenan directamente en la query SQL
  • No hay validación ni escape de caracteres especiales
  • Un atacante puede "romper" la query e insertar su propio código SQL

⚡ Ejemplo de Explotación:

Input del atacante:

  • Usuario: admin' --
  • Contraseña: cualquiera

Query resultante:

SELECT * FROM users WHERE username='admin' -- ' AND password='cualquiera'

Resultado: El comentario -- hace que se ignore toda la verificación de contraseña.

🛡️ Solución Segura:

cursor.execute("SELECT * FROM users WHERE username=? AND password=?", (username, password))

Usar consultas parametrizadas evita completamente esta vulnerabilidad.
